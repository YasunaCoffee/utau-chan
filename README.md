# うたうちゃん

**AIにたのむと、作曲・調教してキャラクターが歌う** — AIに操作させる前提の、ちいさな歌声合成器(UTAU / ボーカロイド風)です。Claude Code 用の起動スキルを同封しているので、AIに丸ごとお任せできます。

人間用の作曲GUIはありません。譜面は「うたテキスト」というプレーンテキスト形式で、AI(や人間)が1行ずつ書けます。声は2通りで鳴らせます:

- **フォルマント合成**（録音なし・依存ゼロ）— 内蔵キャラ(うたこ / ちびすけ / ロボまる / おじさま)がその場で歌う
- **実録音のUTAU音源**（重音テト等）— `singteto.js` が録音サンプルを繋いで歌わせる

書いた譜面は、`index.html` に貼ればブラウザで口パク再生(実録音のUTAU音源フォルダを読み込めば実際の声でも歌えます)、`node sing.js 譜面.uta` でWAV化、URLに埋め込んで共有もできます。

## AIにたのむ(主機能)

このプロジェクトの主な使い方は、AIにたのむこと。このREADME(またはプレイヤー下部の「かきかた」)をAIに見せて、こう頼むだけです:

> うたテキスト形式で「雨の日のカタツムリ」の歌を書いて。@voice ちびすけ、8小節、フレーズの入りにしゃくり、サビの伸ばす音にはビブラートを付けて調教もやって。

AIが譜面を書き、調教まで付けてくれます。返ってきたテキストを `index.html` に貼るか、`.uta` に保存して `node sing.js`(実録音のテトなら `node singteto.js`)で鳴らします。

起動スキルを同封しています(`.claude/skills/run-utau/`)。[Claude Code](https://claude.com/claude-code) なら `/run-utau` と打つだけで、譜面書き→WAV生成→再生までAIが一括でやってくれます。

## 自分で動かす

```bash
git clone https://github.com/YasunaCoffee/utau-chan.git
cd utau-chan
node sing.js songs/kirakira.uta          # フォルマント合成 → songs/kirakira.wav
node singteto.js songs/kirakira.uta      # 重音テトの実録音で → songs/kirakira.teto.wav（音源は別途DL）
```

またはブラウザで `index.html` を開いてサンプルボタン(⭐🐱🐑🌸🤖)を押すだけ。
GitHub Pages(Settings → Pages → main / root)で公開すれば、共有リンクごと誰でも遊べます。

## うたテキスト形式

```
# コメント
@tempo 104              テンポ(BPM)
@voice うたこ            うたこ / ちびすけ / ロボまる / おじさま
@style ふつう            べたうち / ふつう / こぶし / ささやき / げんき

ど4:き ど4:ら そ4:き そ4:ら | ら4:ほ ら4:し そ4:が:8
ふぁ4:よ ふぁ4:ぞ み4:ら み4:に | れ4:ひ れ4:か ど4:る:8~
```

音符は `高さ:歌詞[:長さ][調教記号]` を空白区切りで並べます。

| 要素 | 書き方 |
|---|---|
| 高さ | `ど れ み ふぁ そ ら し` + オクターブ(`ど4` = C4 = 真ん中のド)。`#`/`♭`、`C4` `F#4` 式もOK |
| 歌詞 | かな1音。`ー` のばす、`っ` 息つぎ、`にゃ` `きゃ` など拗音OK |
| 長さ | 16分音符いくつぶんか。省略=4(4分音符)。`ど4:ら:8` = 8×16分=2拍 |
| 休符 | `・:4`(または `R:4`) |
| 小節線 | `\|` は読み飛ばされる飾り。見やすさのためにどうぞ |

### 調教

べた打ちを人間らしく歌わせるピッチ・表情の作り込みを、テキストで指定できます。

スタイル(自動調教) — `@style` で一括指定:

| スタイル | 効果 |
|---|---|
| `べたうち` | 無調教。しゃくり・ビブラート・ゆらぎ全部なし。ロボ感を愛でる用 |
| `ふつう` | ほどよいしゃくり・ビブラート・音程のゆらぎ |
| `こぶし` | 演歌。深いしゃくり、フレーズ末のフォール、長い音に自動でこぶし |
| `ささやき` | ブレス多め・小さめの声・浅いビブラート |
| `げんき` | 明るめの声・強めの抑揚・早めのビブラート |

調教記号(手動調教) — 音符の後ろに付けて1音ずつ指定:

| 記号 | 効果 |
|---|---|
| `<` | しゃくり(下から音程に入る) |
| `>` | フォール(語尾を落とす) |
| `~` | ビブラート強め |
| `_` | ビブラートなし |
| `*` | こぶし(音程の節回し) |
| `!` | アクセント(強く) |
| `?` | よわく |

例: `そ4:ら:8<~` = ソの音2拍、しゃくって入ってビブラート強め。

## かたりモード(ポエトリーリーディング)

`@mode かたり` と書くと、音符なしでふつうの詩をそのまま読んでくれます。

```
@mode かたり
@voice うたこ
@style ささやき

よるのまちを ひとりで あるいていた
ポケットのなかで ゆびさきだけが さむかった

ねえ、きょうのことは
ぜんぶ わすれてもいいけど。
```

- 抑揚は自動生成 — フレーズの入りは高く、終わりに向かって自然に下降し、モーラごとに音程が小さく揺れます
- `、` 小さな間 / `。` `!` 大きな間 / `?` は語尾が上がる / 空行はもっと大きな間 / スペースは息つぎ
- `@tempo` は1分あたりのモーラ数(省略時320。240でゆっくり、400で早口)
- 本文はひらがな/カタカナで(漢字は読み飛ばされます)

組み合わせのおすすめ: `@style ささやき` で囁き系ポエトリー、`@voice ロボまる` + `@style べたうち` で抑揚ゼロの機械よみ、`@voice おじさま` で渋い朗読。

## 実録音の声で歌わせる(UTAU音源 / 重音テト)

`sing.js` は録音を使わないフォルマント合成。対して `singteto.js` は、UTAU音源(録音WAV + `oto.ini`)を切り貼りして、同じうたテキスト譜面を実際の声で歌わせるミニUTAUリサンプラーです。

```bash
node singteto.js songs/kirakira.uta                    # → songs/kirakira.teto.wav
node singteto.js songs/kirakira.uta --bank "<音源フォルダ>"
UTAU_BANK="<音源フォルダ>" node singteto.js songs/kirakira.uta
node singteto.js songs/kirakira.uta --no-vib --dry     # ビブラート/残響オフ
```

- 連続音(VCV)/単独音(CV)を自動判別。連続音なら「前の母音→子音」の遷移録音でつなぎ、先行発声で拍に合わせます。
- 移調はサンプル毎の実測ピッチ基準(YINで基準F0を推定)。音痴になりません。
- フォルマント保持移調(TD-PSOLA)。ピッチを変えても声色が変わらず、チップマンク感が出ません。
- 長い音には自動でビブラート、音のつなぎは等パワークロスフェード、仕上げに軽い残響。
- 音源フォルダの既定探索先は `~/dev/teto-voicebank/**/`(`重音テト連続音` → `重音テト単独音` の順)。

> 音源ファイルは同梱していません。重音テト音源(© 小山乃舞世)は再配布不可のため、[公式ページ](https://kasaneteto.jp/)から各自ダウンロードし、`--bank` か `UTAU_BANK` でフォルダを指してください。個人利用・改変・非商用〜個人の有料配信/広告収益はOK、クレジット表記は不要(公式ガイドライン)。生成した歌声WAV・音源はこのリポジトリに含めていません。

音質改善の過程(音痴→フォルマント保持まで)は [`docs/teto-changelog.md`](docs/teto-changelog.md) に。品質メトリクスは `node tools/analyze-teto.js 歌.wav 譜面.uta` で測れます。

### ブラウザでもUTAU音源で歌わせる

`index.html` から、CLIと同じリサンプラー(`teto-web.js`)で実録音のUTAU音源を鳴らせます。CLI不要・ブラウザだけで完結します。

1. 「📂 音源フォルダを読み込む」を押し、`oto.ini` の入った音源フォルダを選ぶ(フォルダごと選択。`file://` で開いても動きます)
2. 「🎤 UTAUで歌う」を ON にして「うたう」/「WAV保存」

- 読み込んだ音源は **IndexedDB にキャッシュ** され、次に `index.html` を開くと自動で復元されます(毎回選び直す必要なし)。
- 連続音(VCV)/単独音(CV)の自動判別・YIN実測移調・TD-PSOLA・残響までCLIと同一処理。移植は数値的にほぼビット一致で検証済み。
- テト以外のUTAU音源も同じ手順で読み込めます(表示名は選んだフォルダ名になります)。
- 音源はブラウザ内だけで処理され、どこにも送信・保存されません(再配布不可の音源も安心)。

## 本家UTAUへ書き出す(USTエクスポート)

書いたうたテキストは、[本家UTAU](https://utau2008.xrea.jp/)のプロジェクトファイル(UST)に変換して持っていけます。OpenUtauでも開けます。

```bash
node export-ust.js songs/kirakira.uta            # → songs/kirakira.ust (Shift-JIS)
node export-ust.js songs/kirakira.uta --utf8     # UTF-8で保存(OpenUtau向け)
node export-ust.js songs/kirakira.uta -o 出力.ust
```

ブラウザなら `index.html` の「📤 UST書き出し」ボタンで同じものがダウンロードできます。

調教もいっしょに持っていきます:

| うたテキスト | UST |
|---|---|
| `<` しゃくり / ポルタメント | Mode2ピッチベンド(PBS/PBW/PBY) |
| `>` フォール | 音尻のピッチ点 |
| `~` `_` / スタイルのビブラート | VBR |
| `!` アクセント / `?` よわく | Intensity |
| `ー` のばす | 直前の母音のかなに展開 |
| `っ` 息つぎ / `・` 休符 | 休符(R) |

`*` こぶしはUSTに相当機能がないため省略されます。かたりモード(`@mode かたり`)は音符がないため書き出せません。
音源は同梱されないので、UTAUで開いたあとプロジェクトのプロパティで手持ちの音源(重音テト等)を選んでください。エンコードは本家UTAUに合わせて既定Shift-JIS(改行CRLF)です。

## ファイル構成

```
engine.js              うたエンジン(譜面パーサ + フォルマント合成DSP)。ブラウザ/Node両対応
sing.js                CLI: .uta → .wav(フォルマント合成)
singteto.js            CLI: .uta → .teto.wav(UTAU録音音源で歌う。VCV/CV自動判別・PSOLA)
export-ust.js          CLI: .uta → .ust(本家UTAU/OpenUtau用プロジェクトファイル)
index.html             プレイヤー(貼って再生、WAV保存、共有リンク、口パクキャラ、楽譜プレビュー、UTAU音源読み込み)
teto-web.js            ブラウザ版UTAUリサンプラー(singteto.js移植。index.htmlが音源フォルダを読み込んで合成)
songs/*.uta            サンプル譜面(調教スタイルのデモ含む)
tools/analyze-teto.js  歌声の品質メトリクス測定(音程/ビブラート/クリック)
docs/teto-changelog.md 重音テト調教の改善履歴(v1→v5)
.claude/skills/run-utau/  起動用スキル(Claude Code から呼べるランブック)
package.json           メタ情報 / npm scripts(sing・sing:teto・analyze・smoke)
```

すべてルート実行のフラット構成です。`npm run sing`(= `node sing.js`)のように
npm scripts からも呼べますが、従来どおり `node sing.js 譜面.uta` で直接動きます。

## しくみ

フォルマント合成(`sing.js` / `engine.js`)

1. 歌詞のかなをモーラに分解し、子音+母音へ(`きゃ` → k + 硬口蓋わたり + a)
2. 母音はのこぎり波(声帯)を4本のバンドパスフィルタ(声道フォルマントF1〜F4)に通して生成
3. 子音はノイズバースト(か・た行)、摩擦ノイズ(さ・は行)、鼻音ハム(な・ま行)などをタイプ別に合成
4. 調教はすべてピッチ軌道の変形として実装(しゃくり=下からの指数的アプローチ、こぶし=短い装飾振動、ビブラート=遅延付きLFO、ゆらぎ=低速ドリフト)
5. 仕上げにブレスノイズとシュローダー・リバーブ、ピーク正規化

UTAU音源(`singteto.js`)

1. `oto.ini`(Shift-JIS)を読み、かな+直前の母音からサンプルを選ぶ(VCV/CV自動判別)
2. 各サンプルの基準ピッチをYINで実測し、目標音程との比で移調
3. 子音部はそのまま、母音は TD-PSOLA(2周期Hann粒をピッチ同期で重畳)でフォルマント保持したまま伸長+ビブラート
4. 先行発声で拍に合わせ、音のつなぎを等パワークロスフェード、仕上げに残響と正規化

どちらも依存ゼロの素のJavaScriptです。
