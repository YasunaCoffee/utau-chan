# 重音テト調教(singteto.js)改善ログ

`singteto.js` は うたテキスト譜面を UTAU録音サンプル(重音テト等)で歌わせるミニリサンプラー。
その音質改善の履歴を、**同じ物差し**で記録する。

## 測り方(再現可能)

```bash
node tools/analyze-teto.js songs/history/<版>.teto.wav songs/kirakira.uta
```

- **音程誤差**: 各音の実測F0(YIN, 110–500Hzに制限)と目標音程との差(セント)。
  1音を複数点で測り中央値を取りオクターブ誤検出を除去。「中央値」を主指標に、「50c超の音数」を外れ具合として併記。
- **ビブラート**: 0.5秒以上の音でF0が15–130c揺れている数。
- **クリック**: 隣接サンプルの急ジャンプ回数(|Δ|>0.25 が大クリックの目安)。

音のスナップショットは `songs/history/*.teto.wav`(再配布禁止のためgit管理外・ローカル比較用)。

## 履歴(kirakira.uta で測定 / 解析器は音の中心0.3〜0.55で測るv2)

| 版 | 内容 | 音程誤差 中央値/平均/50c超 | ビブラート | クリック>0.25 |
|---|---|---|---|---|
| v1 | CV初版。基準D#4固定・母音tileループ・ビブ無し | 23c / 47c / 16音 | 8 | 161 |
| v2 | CV。**YINでサンプル毎に実測ピッチ移調**・ビブラート・XFループ・軽い残響 | 8c / 17c / 4音 | 24 | **122** |
| v3 | **連続音(VCV)対応**。前母音→子音の遷移録音でつなぐ。先行発声で拍合わせ | 4c / 10c / 2音 | 18 | 836 ⚠ |
| v4 | **TD-PSOLA でフォルマント保持移調**。母音ループ廃止(うなり源を除去) | 6c / 12c / 2音 | 19 | 901 |
| v5 | **接続の作り直し**。前母音を切り詰め前音を次子音直前まで伸長、境界の音量の谷を低減 | 6c / 15c / 3音 | 34 | 944 |

### フォルマント保持(声色が音程で変わらないか / スペクトル重心の低音-高音差)
- v2(リサンプル移調): 差 **1056Hz**(高い音ほど細く明るい=チップマンク感)
- v4/v5(PSOLA): 差 **207Hz**(約1/5。声色がほぼ一定に)

### 接続(境界の音量の谷 / 谷比<0.45の境界数)
- v4: 24/41  → v5: **14/41**(前母音を子音直前まで伸ばした効果)。残りは無声子音の自然な区切り。

### 各版でやったこと

- **v1**(commit `b4e2e9b`): 単独音(CV)対応。`oto.ini`+録音WAVを、固定子音部+母音tileで連結。
  基準ピッチをD#4(MIDI63)固定にしたため、元がD4のサンプル(し・ら等)が約半音低く出た。
- **v2**(commit `09f4a10`): 音程を最優先で修正。**YIN**で各サンプルの基準F0を実測し移調(中央値23→8c)。
  長音にビブラート(±28c/5.7Hz/立ち上がり遅延)、母音持続をクロスフェードループ化、子音↔母音の継ぎ目もXF。
  軽いシュレーダー残響。`--no-vib` / `--dry` で無効化可。
- **v3**(連続音): otoから連続音/単独音を自動判別。エイリアス `<前母音> <かな>`(語頭は `- かな`、
  無ければ母音付きを借りて先頭母音をカット)。先行発声(preutterance)で拍に合わせ、オーバーラップ区間で
  前音と等パワークロスフェード(最長35msに制限)。**音程は最良(中央値4c)で子音遷移も自然**。
  → ユーザー評価: 「つなぎは自然になったが"つなぎ方"に難あり」。境界に音量の谷。
- **v4**(PSOLA): 母音ループを廃し、**TD-PSOLA**(2周期Hann粒をピッチ同期で重畳)で母音を生成。
  フォルマントを保ったまま移調・伸長するので、チップマンク感が激減(重心差1056→207Hz)、ループのうなりも消える。
  周期は実測F0、粒はマーク列をピンポンして持続。ビブラートは目標周期を変調して付与。
- **v5**(接続改): v3/v4で残った境界の音量の谷を修正。**頭の前母音を切り詰め(≤90ms、語頭35ms)**、
  前の音を次の子音直前まで伸ばして重ね、クロスフェードを60msに。深い谷 24→14/41。

### 解析器メモ
- 音程は各音の中心寄り(0.3〜0.55)を複数点測って中央値。後半は先行発声で隣の音にかぶり誤検出する(VCVで顕著)。
- 「クリック」指標は|Δ|の大きさで、明るさ・高域と本物の不連続を厳密には区別しない(生録音は0/秒)。
  真の1サンプルスパイクは全版0。v3の836は主に高域のざらつき(うなり)と推定。

## 残タスク(効きそうな順)

1. **v3のざらつき低減** — 母音ループのピッチ同期化、または重なり由来のうなり対策。要・聴感評価。
2. **フォルマント保持移調(WORLD/PSOLA)** — 現状は伸縮移調でピッチが離れると声色が変わる(チップマンク感)。
3. **ノート間レガート** — 音の変わり目のクロスフェード最適化。
4. 調教記号(しゃくり/フォール)のサンプル側反映。
